CS technical interview preparation advice

Buck’s background

I taught at App Academy. I’ve done onsites at Google, Twitter, Apple, PayPal (on their Stingray team, so the experience probably doesn’t generalize), Uber, and a whole bunch of smaller companies. I passed all these interviews except Uber. I used to teach at App Academy, so I’ve watched lots of people without traditional backgrounds get tech jobs.

What things should you know?

I particularly recommend being able to write the following algorithms pretty much at the speed of typing:

  • Breadth first search through a graph (I swear that at least 20% of my technical interviews involve writing a BFS)
  • Binary search
  • Quicksort (you don’t have to know how to do it in place)

other good resources:

  • Steve Yegge’s classic Get That Job At Google I think slightly exaggerates the difficulty of Google interviews, but is good advice. The sections from Tech Prep Tips through Operating Systems are a pretty good summary of good things to know.
  • Cracking the Coding Interview is great. I think it’s worth reading the solutions even if you don’t want to bother figuring them out yourself.

Questions

Two sum

Given an array of integers and a target number, find the indices of two numbers in the array that they add up to the target. For example, two_sum([1, 2, 7, 3], 5) should return [1, 3].

Initially, a solution with O(n^2) time is okay. Then, do it in O(n) time.

Find the repeated element

You’re given an array which is a permutation of the integers from 0 to n with a single repeat. For example, you might be given [0, 1, 2, 2] or [2, 1, 0, 1] or [4, 2, 1, 2, 3, 0].

Given such an array, return the repeated element. You can do it in O(n) time.

Island perimeter

You’re given an ASCII map, like this:

........
...XXXX.
..XXXXX.
..XXXXX.
...XXX..
........

which has a single island on it. A point on the map is said to be on the perimeter of the island if it’s horizonally or vertically next to the water. Here’s the perimeter of that previous map:

........
...PPPP.
..PXXXP.
..PXXXP.
...PPP..
........

Write a function which takes that map and returns the number of points on the map which are on the island.

Make change

Write a function which takes a list of coin denominations, and a number, and returns the way that you can make that change with the fewest coins. For example, if I had to make 31 cents using American coins, I’d use a quarter, a nickel, and a penny. So make_change([1, 5, 10, 25], 31) returns [25, 5, 1]. Some countries have wonkier denominations which are harder: make_change([1, 7, 10], 9) returns [7, 1, 1], and make_change([1, 7, 10], 15) returns [7, 7, 1].

Maximum sum subarray

Given an array, like [2, 4, -5, -3, 5, 6, 4, -1, 5, -2], return two indices i and j to maximize the sum of the array from i to j. In that case above, it would return [4, 8].

The fastest solution takes O(n) time.
